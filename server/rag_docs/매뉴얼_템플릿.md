# HRC Library 이용 안내

이 문서는 HRC Library 웹 앱 이용 방법을 설명한다. 챗봇(RAG)이 검색·인용할 수 있도록 섹션 단위로 나누어 두었다.

## 용어 정의

- **문항(샘플)**: 설문에 쓸 수 있는 질문 예시 단위. 각 문항은 고유 번호(qnum 등)로 구분된다.
- **카테고리**: 문항이 속한 유형·분류(예: 단일, 다중, 그리드). 사이드바의 Sample 아래 폴더로 표시된다.
- **코드 탭**: 선택한 문항의 QM/Perl 등 소스 코드를 보거나 복사하는 영역이다.

---

## 시작하기 — 첫 화면에서 할 일

사용자는 접속 후 왼쪽 **사이드바**에서 문항을 선택한다. 선택하면 가운데 **미리보기(뷰)**와 오른쪽 **코드** 영역에 해당 문항 내용이 표시된다.

메뉴가 보이지 않으면 상단의 **메뉴(햄버거) 아이콘**으로 사이드바를 연다.

---

## 사이드바 — 공지·Update·Sample

### 공지사항과 Update

- **공지사항**: 클릭하면 **모달**로 공지 목록이 열린다.
- **Update**: 클릭하면 최근 반영된 샘플 목록이 모달로 열린다. 항목에 연결된 문항이 있으면 클릭 시 해당 문항으로 이동한다.

### Sample 목록과 카테고리

- **Sample**을 펼치면 카테고리(폴더) 목록이 나온다. 폴더를 누르면 그 안의 문항 목록이 펼쳐진다.
- **현재 선택한 문항이 속한 카테고리**만 자동으로 펼쳐지는 동작을 기본으로 할 수 있다. 검색 중에는 검색에 걸린 카테고리가 펼쳐질 수 있다.
- 문항 옆에 보이는 **대괄호 `[ ]` 묶음**은 태그(속성 키워드)로, 여러 개가 칩 형태로 표시될 수 있다.

### 매뉴얼·샘플 추가·문의하기

- **매뉴얼**: 클릭 시 안내된 **로컬 폴더 경로**가 클립보드에 복사된다. 파일 탐색기 주소창에 붙여넣어 폴더를 연다.
- **샘플 추가**: 새 샘플 제안 양식을 모달로 연다.
- **문의하기**: 문의 내용을 보내는 모달을 연다.

---

## 검색 — Find Sample

사이드바 상단 **Find Sample** 입력란에 글자를 입력하면 문항 번호·태그·유형 문자열에 대해 **부분 일치**로 필터된다.

- 검색어가 있을 때는 **결과가 있는 카테고리**가 펼쳐져 목록을 한꺼번에 볼 수 있다.
- 검색어를 지우면 다시 **선택한 문항 기준**으로 카테고리 펼침이 동작할 수 있다.

---

## 뷰(미리보기) 영역

선택한 문항의 설문 미리보기가 표시된다. PC/모바일 보기 전환 등 UI는 화면 상단 컨트롤을 따른다. (세부 항목은 제품 버전에 맞게 이 섹션을 채운다.)

---

## 코드 영역

선택한 문항이 있을 때 **QM / Perl** 등 탭과 엔진(문항/조건) 선택이 가능하다. 하단 **Copy**로 에디터 내용을 클립보드에 복사할 수 있다.

---

## 자주 묻는 질문 (FAQ)

**Q: 문항은 어디서 고르나요?**  
A: 왼쪽 사이드바의 **Sample**에서 카테고리를 펼친 뒤 목록에서 문항을 클릭하면 선택됩니다.

**Q: 코드만 보고 싶어요.**  
A: 문항을 선택한 뒤 오른쪽 **코드** 패널에서 탭과 복사 버튼을 사용합니다.

**Q: 매뉴얼 PDF는 어디 있나요?**  
A: 사이드바 **매뉴얼** 버튼을 누르면 클립보드에 복사되는 경로의 폴더를 열어 확인합니다. (경로는 `client/src/config/manualOpenPath.js`에서 관리할 수 있습니다.)

**Q: 서버에 연결이 안 된다고 나와요.**  
A: 상단 노란 안내가 보이면 네트워크·서버 상태를 확인하고 **새로고침**을 눌러 다시 시도합니다.

---

## 주의·제한 사항

- 브라우저는 보안상 **임의의 로컬 폴더를 직접 여는 것**을 막을 수 있어, 매뉴얼은 **경로 복사 후 탐색기에서 열기** 방식을 사용한다.
- 챗봇(RAG)은 **이 문서와 추가로 넣는 매뉴얼 파일**을 근거로 답한다. 문서에 없는 내용은 추측하지 않도록 시스템 프롬프트에서 제한하는 것이 좋다.

## 새 섹션 추가 시 (복사용 블록)

아래 블록을 복사해 `## 제목` 아래에 붙이고 내용만 바꾼다.

```markdown
## (여기에 주제 한 줄)

(이 섹션만 읽어도 의미가 통하도록 3~8문장으로 설명한다.)

- 불릿으로 단계나 조건을 나눌 수 있다.

**Q: (사용자 질문 한 줄)?**  
A: (답변 한 블록 안에서 끝까지 설명한다.)
```
