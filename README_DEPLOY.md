# HRC 라이브러리 배포 및 서버 유지 가이드

이 문서는 서버에 배포 파일을 정리하고, 로그오프 후에도 서버가 계속 꺼지지 않고 실행되도록 설정하는 방법을 설명합니다.

## 1. 배포 파일 준비 (`deploy.bat`)

1. `src` 폴더 안의 `deploy.bat` 파일을 더블 클릭하여 실행하세요.
2. 실행이 완료되면 `src` 폴더 안에 **`deploy`** 라는 새로운 폴더가 생깁니다.
3. 이 `deploy` 폴더 안에 다음 파일들이 모두 모여있는지 확인하세요:
    - `server.exe`
    - `dist` (폴더)
    - `cert.pem`, `key.pem`, `.env` (있는 경우)

> **참고**: 실제 운영 서버로 파일을 옮겨야 한다면, 이 `deploy` 폴더 전체를 복사해서 옮기시면 됩니다.

---

## 2. 서버 계속 켜두기 (서비스 등록)

서버 컴퓨터에서 로그오프를 하거나 창을 닫아도 서버가 죽지 않게 하려면 **Windows 서비스**로 등록해야 합니다.

### 방법 A: NSSM 사용 (강력 추천)
가장 안정적인 방법입니다. 서버가 오류로 죽더라도 자동으로 재시작해줍니다.

1. **NSSM 다운로드**:
   - [https://nssm.cc/download](https://nssm.cc/download) 에 접속하여 최신 버전(예: nssm-2.24.zip)을 다운로드합니다.
   - 압축을 풀고 `win64` 폴더 안의 `nssm.exe` 파일을 `deploy` 폴더 안으로 복사해 넣습니다.
   - 즉, `server.exe`와 `nssm.exe`가 같은 폴더에 있어야 합니다.

2. **서비스 등록 스크립트 실행**:
   - `deploy` 폴더 안의 `install_service_nssm.bat` 파일을 **마우스 우클릭 -> 관리자 권한으로 실행** 하세요.
   - "서비스가 성공적으로 등록되고 시작되었습니다!" 메시지가 나오면 성공입니다.

3. **확인**:
   - 윈도우 키 + R -> `services.msc` 입력 -> 확인.
   - 리스트에서 `HRC_Library_Server`를 찾아 상태가 '실행 중'인지 확인하세요.

---

### 방법 B: 작업 스케줄러 사용 (다운로드 불필요)
추가 프로그램 설치가 곤란할 때 사용하세요.

1. `deploy` 폴더 안의 `register_task_scheduler.bat` 파일을 **마우스 우클릭 -> 관리자 권한으로 실행** 하세요.
2. 등록이 완료되면 컴퓨터를 **재부팅** 해야 서버가 켜집니다.

---

## 3. 서비스 삭제 / 중지 방법

서버를 업데이트하거나 삭제하고 싶을 때 사용하세요.

**NSSM 사용 시 (관리자 cmd 창에서 입력):**
```cmd
nssm stop HRC_Library_Server
nssm remove HRC_Library_Server confirm
```

**작업 스케줄러 사용 시 (관리자 cmd 창에서 입력):**
```cmd
schtasks /delete /tn "HRC_Library_Server" /f
```
