# 행정구역(법정동) 10자리 코드 — 자리 규칙과 분리 방법

행정기관·통계에서 쓰이는 **10자리 숫자 코드**를 시도 / 시군구 / 읍면동(및 리) 단위로 해석하는 기준과, 클라이언트 유틸 `parseAdminDistrictCode10` 사용법을 정리한 문서입니다.

---

## 1. 배경과 공식 근거

- **행정안전부**는 **행정표준코드**를 정의·관리하며, 그중 **법정동코드**가 대표적으로 **10자리 숫자** 형식입니다.
- 코드표·변경 이력은 **행정표준코드관리시스템**([https://www.code.go.kr](https://www.code.go.kr))에서 확인할 수 있습니다.
- 공공데이터포털 등에도 「행정안전부_행정표준코드_법정동코드」 등 동일 체계의 데이터가 배포됩니다.

이 문서의 **자리 분할 규칙(2+3+3+2)**은 위 행정표준코드(법정동) 체계와 통계청 등에서 쓰는 **통계용 행정구역코드**(앞 5자리·8자리 시군구·읍면동 코드) 표기와 일치합니다.

---

## 2. 자리수별 의미(일반 규칙)

10자리를 **왼쪽부터** 다음처럼 나눕니다.

| 위치(1-based) | 길이 | 의미 |
|---------------|------|------|
| 1–2 | 2 | **시도코드** |
| 3–5 | 3 | **시군구코드** (해당 시도 내 3자리) |
| 6–8 | 3 | **읍면동코드** (해당 시군구 내 3자리) |
| 9–10 | 2 | **리코드** (리·통 등 하위; 없으면 `00`이 많음) |

### 2.1 실제 데이터 예시

| 행정기관코드(10자리) | 시도명 | 시군구명 | 읍면동명 | 분해 (2\|3\|3\|2) |
|----------------------|--------|----------|----------|-------------------|
| 1111051500 | 서울특별시 | 종로구 | 청운효자동 | `11` \| `110` \| `515` \| `00` |
| 1111053000 | 서울특별시 | 종로구 | 사직동 | `11` \| `110` \| `530` \| `00` |
| 1111054000 | 서울특별시 | 종로구 | 삼청동 | `11` \| `110` \| `540` \| `00` |
| 1111055000 | 서울특별시 | 종로구 | 부암동 | `11` \| `110` \| `550` \| `00` |
| 1111056000 | 서울특별시 | 종로구 | 평창동 | `11` \| `110` \| `560` \| `00` |

- 앞 **5자리** `11110`은 서울(11) + 종로구(110)를 합친 **시군구코드(5자리)**로 통계·DB에서 자주 씁니다.
- 앞 **8자리**는 **읍면동까지** 포함한 코드(통계상 읍면동 단위)로 씁니다.

---

## 3. 시도 / 시군구 / 읍면동 “코드”를 어떻게 저장할지

같은 행정구역도 **표기 관례**에 따라 두 가지를 구분합니다.

### 3.1 구간만 자른 코드(계층 내부 일련)

- **시도코드(2)**: 문자열의 1–2번째 자리.
- **시군구코드(시도 제외 3)**: 3–5번째 자리.
- **읍면동코드(시군구 제외 3)**: 6–8번째 자리.
- **리코드(2)**: 9–10번째 자리.

예: `1111051500` → 시도 `11`, 시군구(3) `110`, 읍면동(3) `515`, 리 `00`.

### 3.2 접두(누적) 코드 — 통계·API에서 흔함

- **시도**: 앞 2자리 → `11`
- **시군구(5자리)**: 앞 5자리 → `11110`
- **읍면동(8자리)**: 앞 8자리 → `11110515`
- **전체 10자리**: 리까지 포함한 법정동 단위 코드

DB나 API에서 “시군구코드”가 **5자리**로 오는 경우가 많은 이유가 여기에 해당합니다.

## 4. 항상 이 규칙을 쓸 수 있는지, 예외

| 상황 | 설명 |
|------|------|
| **법정동 10자리·동일 체계의 행정구역코드** | 자리 의미는 **고정**이다. 이 문서의 분해 규칙을 적용한다. |
| **10자리가 아닌 코드** | 7자리 등 다른 스키마는 **이 규칙을 적용하면 안 된다**. |
| **행정기관코드 vs 법정동코드** | 출처에 따라 행정동용 코드가 법정동과 **항상 같지 않을 수 있다**. odcloud 등 외부 데이터는 필드 정의·샘플로 **동일 10자리 체계인지** 확인할 것. |
| **통·폐합·폐지** | 코드 **값**은 바뀔 수 있으나, **자리 구조(2+3+3+2)**는 유지된다. |
| **시도 해석** | `11`=서울 등은 **별도 시도 코드표**가 필요하다(세종 등 특례 포함). |

## 5. 구현(클라이언트)

### 5.1 모듈 위치

- `client/src/utils/parseAdminDistrictCode10.js`

### 5.2 API 요약

- `parseAdminDistrictCode10(code, options?)`
  - `code`: 문자열 또는 숫자. 공백·하이픈 등은 숫자만 추출 후 처리.
  - `options.strict === true`: 입력 문자열이 **정확히 10자리 숫자**일 때만 성공(앞자리 0 유지).
  - 성공 시 필드:
    - `ok: true`, `raw` (정규화된 10자리)
    - `sidoCode2`, `sigunguCode3`, `sigunguCode5`, `eupmyeondongCode3`, `eupmyeondongCode8`, `riCode2`
  - 실패 시 `ok: false`, `error` 코드, 빈 문자열 필드.

- 상수: `ADMIN_DISTRICT_CODE_LENGTH` (= 10)

### 5.3 사용 예시

```javascript
import { parseAdminDistrictCode10 } from '../utils/parseAdminDistrictCode10';

const r = parseAdminDistrictCode10('1111051500');
if (r.ok) {
  console.log(r.sidoCode2, r.sigunguCode5, r.eupmyeondongCode8);
  // 11, 11110, 11110515
}
```

## 6. CodeBook UI·엑셀 연동

라이브러리 **주소 CodeBook**(행정동·법정동) 표는 `client/src/components/AddressCodebookSection.jsx`에서 렌더링한다.  
**행정기관코드·법정동코드**(10자리) 다음에, `parseAdminDistrictCode10`으로 **구간만 자른** 시도·시군구·읍면동 코드 열을 이름 열과 짝지어 둔다. **리코드(말단 2자리)**는 표·엑셀에 노출하지 않는다.

| 순서 | 열 라벨 | 의미 |
|------|---------|------|
| 1 | 행정기관코드 / 법정동코드 | 원본 10자리(raw, 변경 없음) |
| 2 | 시도코드 | 서버 부착 **`disp_sido`**(1–17 표시 체계). 없으면 클라이언트가 통계 10자리 앞 2자리로 폴백 |
| 3 | 시도명 | 서버·스냅샷 라벨 |
| 4 | 시군구코드 | **`disp_sigungu`**(ruleset **v3**: 시도 `d`에 대해 `1000*d+1`~`1000*d+999` 구간에서 시군구 **라벨** 오름차순. 문자열 길이는 시도 1–9면 보통 4자리, 10–17이면 5자리). 없으면 통계 구간 폴백 |
| 5 | 시군구명 | 서버·스냅샷 라벨 |
| 6 | 읍면동코드 | **`disp_eup`**(ruleset **v3**: `int(disp_sigungu)*100+순번`을 **6자리** 고정, 예 100101–100199). 없으면 통계 구간 폴백 |
| 7 | 읍면동명 | 서버·스냅샷 라벨 |

- **10자리가 아니거나** 숫자만 추출했을 때 10자리가 되지 않으면 구간 코드 열은 비우고, 셀에는 `—`로 표시된다(필터·엑셀에서도 빈 값으로 집계).
- **엑셀 다운로드**: **기준연월**, 행정기관코드(또는 법정동코드), **§6** 표의 시도코드·시도명·시군구코드·시군구명·읍면동코드·읍면동명만 포함한다(`rowsToCodebookExportRows`). **`통계10자리_*` 열은 넣지 않는다.** 표시용 규칙은 [codebook-display-code-system.md](./codebook-display-code-system.md) §9.

표 컬럼·분해 규칙·엑셀 헤더를 바꿀 때 아래를 **한 세트**로 맞춘다.

| 작업 | 파일 / 문서 |
|------|-------------|
| 열 순서·라벨·필터·셀 값·엑셀 컬럼 | `client/src/components/AddressCodebookSection.jsx` — `COL_KEYS`, `getCellValueForCol`, `colLabels`, `rowsToCodebookExportRows`, 빈/필터 행 `colSpan={COL_KEYS.length}` |
| 10자리 **자리 규칙** 자체가 바뀌는 경우(공표 변경 등) | `client/src/utils/parseAdminDistrictCode10.js` 및 이 문서 **§2·§4** |
| 이 문서의 표·체크리스트만 바뀐 경우 | `client/docs/address/admin-district-code-10.md`(본 절·§6) |
| 행정·법정 odcloud CodeBook 스냅샷·배치 | `server/docs/codebook-odcloud-snapshot-architecture.md` — 15097972 / 15099158, `GET/POST .../odcloud-codebook` 및 `.../legal` |
| 법정동 **월 배치 SQLite** 코드북(`/api/address/codebook?kind=legal`) | CodeBook UI는 odcloud 법정 스냅샷을 쓰므로 별도; 구역 히스토리만 해당 시 `server/docs/district_codes_operations.md` |
