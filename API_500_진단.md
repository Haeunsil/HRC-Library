# API 에러 진단 가이드

`api_proxy.aspx` 또는 API 호출 시 **500** 또는 **404** 에러가 발생하는 경우 아래를 확인하세요.

---

## 1. 요청 흐름

```
브라우저 → api_proxy.aspx (IIS) → FastAPI (127.0.0.1:8000) → SQL Server DB
```

500 에러는 **api_proxy.aspx** 또는 **FastAPI 백엔드** 중 한 곳에서 발생합니다.

---

## 2. 확인 사항

### 2-1. Python 백엔드 실행 여부

**webdemo.hrcglobal.com** 서버에서 FastAPI가 8000 포트로 실행 중인지 확인합니다.

```powershell
# 서버에서 실행 중인 프로세스 확인
netstat -ano | findstr :8000
```

- 결과가 없으면 → **백엔드가 실행되지 않은 상태**입니다.
- `main.py` 또는 빌드된 exe를 실행해 8000 포트에서 서버를 띄워야 합니다.

```powershell
# exe 빌드: server 폴더에서 build_exe.bat 실행
# 실행: server.exe (server 폴더 또는 배포 경로에 복사 후)
cd server
.\server.exe
# 또는 Python으로 직접 실행
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

---

### 2-2. api_proxy.aspx 에러 내용 확인

500 응답 시 **응답 본문**에 상세 에러가 포함됩니다. 브라우저 개발자 도구 → Network → 해당 요청 → Response 탭에서 확인하세요.

| 에러 내용 | 원인 |
|----------|------|
| `Connection refused` | 백엔드(127.0.0.1:8000)가 실행되지 않음 |
| `Unable to connect` | 백엔드 미실행 또는 방화벽 차단 |
| `Remote Response: ...` | FastAPI에서 반환한 에러 메시지 (DB 등) |

---

### 2-3. 데이터베이스 연결

FastAPI가 실행 중이어도 DB 연결 실패 시 500이 발생합니다.

- **database.py** 환경변수 확인: `DB_SERVER`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`
- 서버에서 DB 서버(10.0.11.175:15000) 접근 가능 여부 확인
- ODBC 드라이버 설치 여부 확인 (ODBC Driver 17/18 for SQL Server)

```powershell
# DB 연결 테스트 (서버에서)
python -c "from database import get_db_connection; c=get_db_connection(); print('OK'); c.close()"
```

---

### 2-4. api_proxy.aspx 위치

- **배포 구조**: 모든 배포 파일은 `HRClib\dist` 안에 있습니다.
- 경로: `https://webdemo.hrcglobal.com/HRClib/dist/api_proxy.aspx`
- `api_proxy.aspx`는 **HRClib\dist** 폴더에 있어야 합니다.
- IIS에서 해당 경로가 올바른 애플리케이션으로 매핑되어 있는지 확인합니다.

| # | 확인 | 조치 |

## 4. 404 Not Found (문의하기/샘플추가 실패 시)

**증상**: `Remote Response: {"detail":"Not Found"}` 또는 404 오류

**원인**: 8000 포트에서 실행 중인 서버에 `/api/inquiry`, `/api/add_question` 라우트가 없음.

**해결**:

1. **최신 서버 빌드·배포**
   ```powershell
   cd d:\esha\3. KPI\2. 라이브러리\src\server
   build_exe.bat
   # 생성된 server.exe를 O:\WebSurveyDEMO_test\HRClib\ 에 복사
   ```

2. **서버 재시작**
   ```powershell
   # 기존 프로세스 종료 후
   cd O:\WebSurveyDEMO_test\HRClib\
   .\server.exe
   ```

3. **라우트 확인**
   ```powershell
   curl http://127.0.0.1:8000/api/get_init
   curl -X POST http://127.0.0.1:8000/api/inquiry -H "Content-Type: application/json" -d "{\"email\":\"test@test.com\",\"message\":\"test\"}"
   ```
   - `get_init`은 응답하는데 `inquiry`만 404 → 서버가 구버전입니다. 최신 `main.py`로 교체 후 재시작하세요.

4. **api_proxy.aspx 최신 버전 배포**  
   `server/api_proxy.aspx`를 `O:\WebSurveyDEMO_test\HRClib\`(또는 dist)에 복사해 최신 버전이 적용되었는지 확인하세요.
